Project Background

This project entails building an AI-powered Information Retrieval Engine that can carry out information retrieval using noisy documents where documents are generated via transcription that can introduce errors. The work entails initial research into existing methods of robust information retrieval and solutions proposed for similar problems, preparing a research report from the findings, identifying the requirements, and proposing a solution application that can carry out information retrieval with noisy data, and finally, development and deployment of the proposed system with necessary integrations.

Software Engineering

MUST
  • Develop an API to allow "plugging in" different search algorithms
  • Have all the libraries be open source
  • Create a Google-like web user interface (UI) to search for content that interacts with the API to display relevant search results
SHOULD
  • Show more interesting information about each material: summary of the contents, thumbnail, author/source information. in the search results
  • User account functionalities
  • Make the interface responsive
COULD
  • Display user interests/ statistics in the interface with charts
WON'T
  • Allow users to create and share watchlists
  • Advanced search interface
  • Show users recommendations based on their history

Machine Learning

MUST
  • Have an algorithm able to retrieve the topic of text files/transcripts and accounting for the noisy nature of transcripts.
  • Respect GDPR and other ethical rules
  • Create a search model/algorithm
  • Use elastic search for the search algorithm
  • Have all the libraries be open source
  • Test our search and topic modelling algorithms through statistical analysis (Benchmarking)
SHOULD
  • Test the search result quality through a survey
  • Test the search results on relevance-labeled data sets
  • Improve time efficiency on the base models/algorithms
COULD
  • Have a better content representation that accounts for transcription errors
  • Use deep learning /a neural model to improve the topic modeling
  • Save a part of the data set to test the accuracy of the topic modeling algorithm against manually labeled data
